# KAIRI-Skills Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------- |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within KAIRI-Skills, please follow these steps:

### 1. Do Not Report on Public Issue Tracker

**Please report security issues privately** to avoid exposing potential vulnerabilities before they're patched.

### 2. Report Directly

Send your report via:

- **Email**: [security@kairi-skills.io](mailto:security@kairi-skills.io)
- **Private GitHub Security Tab**: Use GitHub's security advisories feature

### 3. What to Include

Your report should include:

- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce
- Proof-of-concept or exploit code (if possible)
- Impact of the issue (including how it might affect users)

### 4. What to Expect

- **Acknowledgment**: You'll receive acknowledgment within 48 hours
- **Initial Response**: We'll aim to send an initial response within 72 hours
- **Status Update**: We'll keep you informed of the progress
- **Resolution**: We'll work to release a patch and public disclosure

### 5. Disclosure Policy

We support responsible disclosure and will:

- Credit reporters in the security advisory (with permission)
- Provide early access to fixes before public release
- Follow CVE guidelines for major vulnerabilities

## Security Best Practices

When using these security skills:

1. **Always verify** before running any security tool
2. **Use isolated environments** for testing
3. **Follow legal guidelines** - only test systems you own or have permission to test
4. **Document all findings** for compliance purposes
5. **Keep tools updated** to the latest versions

---

## Tool Safety Guidelines

### Network Analysis Tools
- Only use on networks you own or have explicit permission to monitor
- Protect captured data with encryption
- Follow data retention policies

### Malware Analysis Tools
- Always analyze in isolated, sandboxed environments
- Never analyze malware on production systems
- Follow biosafety level guidelines

### Exploitation Tools
- Only use for authorized penetration testing
- Obtain proper legal authorization
- Document all testing activities
